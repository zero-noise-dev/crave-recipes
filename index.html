Privacy Policy

Last updated: 1 June 2026

Crave Recipes is designed from the ground up to respect your privacy. This policy explains what data the app handles and how.

Data We Collect

Nothing. Crave Recipes does not collect, transmit, or store any personal data on any server. There are no analytics, no crash reporting, no usage tracking, and no advertising identifiers.

We do not collect any data. The table below reflects this — all categories are empty by design.

Data Stored on Your Device

All app data is stored exclusively in your device's local storage. This includes:

  • Recipe content (title, ingredients, steps, notes)
  • Recipe photos you add
  • Tags and cookbook attributions
  • Your Claude API key (used only to make API calls from your device)
  • App preferences and settings

This data never leaves your device except when you explicitly export a backup file.

Claude API Key

When you use an import feature, your device sends recipe content directly to Anthropic's API using your personal API key. This request is made from your device — not through any Crave Recipes server. Please review Anthropic's Privacy Policy for details on how they handle API requests.

Treat your API key like a password. If you share a backup file with others, be aware it will contain your API key — remove it from Settings first if you do not want to share it.

No Account Required

Crave Recipes does not require an account, email address, or any form of registration.

Third-Party Services

The only third-party service Crave Recipes communicates with is the Anthropic Claude API, and only when you actively use an import feature. No advertising networks, analytics services, or other third-party SDKs are used.

Data Deletion

All your data can be deleted at any time by clearing the app's data or uninstalling it. There is no server-side data to delete.
